# M9.2-S7: Framework/Instance Split + Audit Fixes — Sprint Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** (1) Move agentic behavior from `.my_agent/` to repo-root `skills/` so framework changes propagate automatically. (2) Fix all critical and medium findings from the deep agentic flow audit.

**Why:** The tool rename (`create_task` → `create_automation`) broke delegation because framework behavior lived in an unversioned location. Additionally, the audit found stuck skill filters, stale test assertions, duplicate prompt loading, and stale instance skill copies — all silent failures.

**Audit reference:** Risk analysis from the M9.2 planning session (2026-04-07). Findings numbered #1-#15.
