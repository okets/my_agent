# M9.2-S7: Framework/Instance Split — Sprint Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Move agentic behavior (delegation protocol, routing, tool instructions, operational rules) from `.my_agent/` (gitignored, instance-specific, unsyncable) to repo-root `skills/` (version-controlled, framework-owned). Instance keeps identity, personality, trust, and preferences.

**Why:** The tool rename (`create_task` → `create_automation`) broke delegation because framework behavior lived in an unversioned location. The framework updated, the instance didn't. This split ensures framework changes propagate automatically.

**Principle:** If the framework renames a tool, changes a protocol, or updates operational rules, the brain picks it up on next restart. No manual syncing. No stale copies.

## Risk Mitigation

| Risk | Mitigation |
|------|-----------|
| Framework skills duplicate instance content | Step 8 deletes stale instance files. Step 10 checks for no duplicates. |
| Prompt grows too large | The split doesn't add content — it moves it. Total prompt size should stay ~44K chars. |
| Hatching creates stale copies again | Step 9 verifies hatching doesn't copy from the new location. |
| Other instances break | Framework skills are additive. Existing `.my_agent/` files still work until manually cleaned. |
| Test expectations reference `create_task` | Step 11 updates regression tests. |
