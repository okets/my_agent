# M9.2-S5.1: Remove Haiku Visual Fallback — Sprint Plan

> **Branch:** `experiment/remove-haiku-fallback` (off master, after S5 merge)

**Goal:** Remove the Haiku visual augmentation fallback entirely. The S5 skill rewrite + required descriptions should make the brain generate charts proactively. If charts disappear in testing, revert the branch.

**Hypothesis:** After S5's decision tree skill rewrite, the brain calls `create_chart` itself for data-rich responses. The Haiku fallback is now redundant — it fires rarely and when it does, it produces inferior results (separate message, Haiku quality, generic "data chart" descriptions before S5).

**Scope:** Conversation Nina's post-response hook only. Worker chart generation (research template todo item) is unaffected.

**Rollback:** `git checkout master` — the fallback is restored immediately.
