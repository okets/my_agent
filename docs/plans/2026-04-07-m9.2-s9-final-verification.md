### Step 0: Wire skill filter into prompt assembly (S8 gap fix)

S8 made `filterSkillsByTools()` a pure function (returns disabled skill names, no disk writes). But nothing uses the result — skill filtering is computed but never applied. This step completes the wiring.

- [ ] **0a: Add `excludeSkills` parameter to `assembleSystemPrompt()`**

In `packages/core/src/prompt.ts`, add an optional parameter to `assembleSystemPrompt()`:

```typescript
export async function assembleSystemPrompt(
  brainDir: string,
  options?: { excludeSkills?: Set<string> },
): Promise<string> {
```

In the framework skills scan loop (currently lines ~604-618), skip entries whose name (without `.md`) is in the exclude set:

```typescript
for (const entry of entries) {
  if (!entry.endsWith('.md')) continue
  const skillName = entry.replace(/\.md$/, '')
  if (options?.excludeSkills?.has(skillName)) continue
  // ... existing loading logic
}
```

- [ ] **0b: Wire callers to pass disabled skills**

In `packages/dashboard/src/agent/session-manager.ts` (line ~331) and `packages/dashboard/src/automations/automation-executor.ts` (line ~176):

Pass the `filterSkillsByTools()` result into `assembleSystemPrompt()`:

```typescript
const disabledSkills = await filterSkillsByTools(agentDir, sessionTools);
const prompt = await assembleSystemPrompt(brainDir, {
  excludeSkills: new Set(disabledSkills),
});
```

- [ ] **0c: Remove dead `filterSkillsByTools` call in `app.ts`**

In `packages/dashboard/src/app.ts` (line ~1612), the return value is discarded. Remove the call entirely — it does nothing here.

- [ ] **0d: Unit test for exclusion**

Add a test to `packages/core/tests/prompt-always-on.test.ts` (or a new file):

```typescript
it("excludes skills in the excludeSkills set", async () => {
  const prompt = await assembleSystemPrompt(brainDir, {
    excludeSkills: new Set(["visual-presenter"]),
  });
  expect(prompt).not.toContain("Visual Expression");
  expect(prompt).toContain("Message Routing"); // other skills still loaded
});
```
