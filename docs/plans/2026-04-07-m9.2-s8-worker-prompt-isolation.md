# M9.2-S8: Worker Prompt Isolation + Skill Filter Safety — Sprint Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** (1) Workers get a worker-specific prompt — not the full brain prompt with "you do not do work yourself." (2) Skill filter stops writing to disk — runtime filtering prevents stuck `disable-model-invocation` flags from crashed sessions.

**Why:** The audit found that `buildWorkingNinaPrompt()` calls `assembleSystemPrompt()` which gives workers the brain's identity, contacts, daily logs, delegation instructions, and a directive NOT to do work. Workers are told not to do the thing they exist to do. Additionally, the skill filter modifies SKILL.md files on disk, which is inherently fragile — any crash leaves skills permanently disabled.

### Task 1: Worker Prompt Isolation

Workers need: task instructions, todos, tool reference, autonomy level, workspace path.
Workers do NOT need: brain identity, contacts, daily logs, delegation protocol, conversation role, trust tiers, notebook tree.

- [ ] **Step 9: Refactor `filterSkillsByTools` to runtime-only**

In `packages/core/src/skill-filter.ts`:

Replace the disk-write approach with a runtime filter:

```typescript
// Before (dangerous):
// Reads SKILL.md, writes disable-model-invocation: true to disk
// Cleanup in finally block — crashes bypass it

// After (safe):
// Returns a list of skill paths to EXCLUDE from prompt assembly
// No disk writes. Prompt assembly skips excluded skills.
// OR: returns modified skill content strings (with the flag set)
//     without writing to disk
```

The exact approach depends on how skills are loaded. Two options:

**Option A (exclude list):** `filterSkillsByTools()` returns a `Set<string>` of skill paths to skip. Pass this set to `assembleSystemPrompt()` which skips those paths during the skills scan.

**Option B (in-memory modification):** `filterSkillsByTools()` returns modified skill content strings with the flag set, which are used for the session only. No disk writes.

Option A is simpler. The skill scan in `prompt.ts` (lines 644-663) would check the exclude set before loading each skill.
